OrderFlow-Lite

Express + MySQL order processing API, with an internal background worker.

API running

Endpoints

GET /health Liveness probe
GET /ready Readiness probe (checks MySQL)
POST /orders Create an order requires x-api-key
GET /orders List orders requires x-api-key
GET /orders/:id Get an order + its event history requires x-api-key

Quick start

Every /orders request requires an x-api-key header.

See README.md in the repo for the full API reference and local setup.
